基于 Hugo 与 GitHub 的静态网站创建与部署技术手册

本文档用于搭建一个基于 Hugo 的静态博客，并实现与 GitHub 的自动化部署。后续只需专注于内容创作（编写 Markdown 文件），并通过简单的命令即可发布更新。

一、环境准备 #

1、背景说明 #

  • Hugo：一款用 Go 语言编写的快速、现代的静态网站生成器。它能将内容源文件（如 Markdown）转换为静态 HTML 和 CSS 文件。静态网站是由固定的 HTML 文件组成，无需数据库支持，加载速度快，安全性高，非常适合博客和个人站点。

  • GitHub Pages：GitHub 提供的一项服务，允许用户通过仓库托管静态网页，实现外网访问。

2、前置工具安装 #

在正式创建网站前，请确保本地环境已安装以下工具：

  • Git：版本控制工具，用于管理代码和同步到 GitHub。可下载Git for Windows版本。

  • Hugo：下载 Windows 版本的二进制文件（推荐下载包含 Sass/SCSS 支持的 extended 版本，如 hugo_extended_x.x.x_windows-amd64.zip），并将其解压路径添加到系统的 Path 环境变量中，以便在终端全局调用。

二、Git 与 GitHub 环境配置 #

1、Git 基础配置 #

安装 Git 后，在 Windows 系统上打开 Git Bash，执行以下命令设置全局用户名和邮箱：

git config --global user.name '用户名'
git config --global user.email '登录邮箱'
  • 提示：可通过 git config --list 查看当前所有配置。

2、配置 SSH 密钥 #

为了实现本地仓库与 GitHub 的免密通信，需要生成并配置 SSH 密钥对：

  1. 生成密钥：在终端执行 ssh-keygen -t rsa -C "登录邮箱"。

  2. 获取公钥：该命令会在 ~/.ssh/ 目录下生成 id_rsa（私钥）和 id_rsa.pub（公钥）。复制公钥文件内的所有内容。

  3. 添加到 GitHub：登录 GitHub，进入 Settings -> SSH and GPG keys，新建 SSH key，粘贴公钥内容。

  4. 测试连接：终端执行 ssh -T git@github.com。若出现欢迎提示信息，则代表连接成功。

3、常见连接问题排查 (Debug) #

如果在连接时出现 ssh: connect to host github.com Port 22: Connection refused 错误，通常是因为网络端口限制。需要在 ~/.ssh/ 目录下创建或修改 config 文件，添加以下内容以强制使用 Port 443：

参考链接1，参考链接2

Host github.com  
User 登录邮箱
Hostname ssh.github.com  
PreferredAuthentications publickey  
IdentityFile ~/.ssh/id_rsa  
Port 443

三、Hugo 网站创建与本地开发 #

1、初始化项目 #

# 创建一个名为BBlog的项目
hugo new site BBlog

# 进入项目目录，并初始化本地仓库。
cd BBlog 
git init

2、集成主题 #

Hugo拥有丰富的主题生态，本次选用一款功能丰富且易于定制的主题——Congo。

  1. 通过Git下载主题
git submodule add -b stable https://github.com/jpanther/congo.git themes/congo

  1. 配置主题：

    • 删除 Hugo 默认生成的 config.toml 文件。

    • 将 themes\congo\config 文件夹复制到项目主目录下。

    • 打开 config_default\config.toml 文件，在首行添加 theme = "congo"。

3、本地预览与构建 #

  • 实时预览：在项目根目录执行 hugo serve。终端会显示本地访问地址（通常是 http://localhost:1313），浏览器打开即可实时查看网站效果。

  • 生成静态文件：执行 hugo --buildDrafts。该命令会在项目根目录下生成 public 文件夹，里面包含了所有可供发布的静态 HTML 文件。

4、部署到 GitHub #

4.1 关联远端仓库 #

在 GitHub 上创建一个新的空仓库，命名为 “Github用户名”.github.io。

关联仓库：在本地 public 目录下执行以下命令：

git init
git remote add origin <项目ssh地址>

注意：GitHub 的默认分支已从 master 改为 main。若需切换分支名，可执行 git branch -m master main。

4.2 首次推送与日常更新 #

首次推送：

git add .
git commit -m "Initial commit"
git push -u origin main

  • 日常更新流程：

    1. 在本地修改文章或配置。

    2. 重新生成静态文件：hugo --buildDrafts。

    3. 进入 public 目录，执行 git add .、git commit 和 git push 同步更新。

5、 主题配置与更新 #

Congo 主题的详细配置参考作者的Demo文档

当 Congo 主题有新版本发布时，可在项目根目录执行以下命令更新主题 submodule：

git submodule update --remote --merge
