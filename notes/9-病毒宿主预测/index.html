病毒宿主预测

宿主预测有四种基本思路：

  • 1）CRISPR比对；

  • 2）tRNA比对；

  • 3）全基因序列比对；

  • 4）寡核苷酸比对

此外，陆续出现各种综合性软件，使用一种或多种方法并结合机器学习预测宿主

  • 传统预测思路
方法 原位 外部 软件
CRISPR MAGs RefSeq database CASC(CRT)、crass
tRNA MAGs tRNAviz database tRNAscan-SE、aragorn
BLASTn MAGs RefSeq database BLAST
ONF MAGs GenBank database VirHostMatcher
  • 外部宿主预测
软件 原理 数据库 调用软件
VirHostMatcher-Net CRISPR与ONF 有 CRT、WIsH
VPF-Class 病毒蛋白家族 有 -
iPHoP CRISPR、ONF等 有 RaFAH、WIsH、VHM、PHP
  • 原位宿主预测
软件 原理
CASC(CRT) CRISPR match
tRNAscan-SE tRNA match
blastn seqence match
VirHostMatcher ONF(k-mer)

9.1 CRISPR对比 #

1. CRISPR的识别 - CASC #

CASC是“CASC Ain’t Simply CRT”，基于CRISPR识别工具CRT(CRISPR Recognition Tool, 2007)

此外，还有基于未组装的reads的识别工具crass

  1. 安装：
git clone https://github.com/dnasko/CASC.git
perl Makefile.PL PREFIX=/Path/to/where/to/install
make
make test
make install
# 写入环境变量（环境变量设置参考https://www.cnblogs.com/yaoqingzhuan/p/10889718.html）
vim ~/.bashrc
PATH=$PATH:/Path/to/where/to/install/bin
export PATH
# export PERL5LIB=/Path/to/where/to/install/lib/perl5/site_perl
export PERL5LIB=/Path/to/where/to/install/lib:${PERL5LIB}
激活环境变量
source ~/.bashrc

debug：在工作目录创建软链接指向$PATH/CASC/bin/CRT1.2b.jar

ln -s ~/software/CASC/bin/CRT1.2b.jar CRT1.2b.jar
  1. 使用：
casc -i TestSeqs.nuc.fasta -o outdir -n 8

参数：-n 线程；--conservative（默认为--liberal）

conservative为CRISPR基因附近带有cas蛋白基因，liberal无

  1. 结果：

输出结果为7个文件

预测MAGs(所有样品在一个文件)的CRISPR序列

选择spaces序列文件，去除少于3个CRISPR的序列

2. blastn比对 #

将筛选后的CRISPR序列与viral contigs序列进行blastn

对比结果选择 identity ≥95% 且 SNPs ≤2 的，确定病毒宿主

9.2 tRNA对比 #

tRNAviz数据库

1. tRNA识别 - tRNAscan-SE（bioconda安装） #

# 分别用两种模式(-A/-B)识别tRNA
tRNAscan-SE -B -a MAGs_tRNA_out.fasta -m MAGs_tRNA.stat MAGs.fasta -q --thread 14

参数：-A 古菌；-B 细菌； 默认真核

-a 输出tRNA序列；-m 输出统计结果；-q 安静输出

另外一个tRNA预测软件ARAGORN，速度非常快，但注意结果的tRNA的编号会重新编排

aragorn -t -fons -o viral_tRNA.fasta viral_blastn_5000.fasta

按MAG的文件分别运行，完成后添加前缀，再合并所有MAG的tRNA序列

2. blastn对比 #

分别预测完MAGs和viral contigs的tRNA后，用blastn对比tRNA序列

# 用MAGs的tRNA序列建库
makeblastdb -in MAGs_tRNA_A.fasta -dbtype nucl -out db/MAGs_A
# 对比
blastn -query viral_tRNA_A.fasta -out blastn_A.out -db db/MAGs_A -outfmt "6 qseqid sseqid qcovs pident length mismatch evalue bitscor
e qlen slen" -num_threads 12

3. 统计数据 #

只筛选完全匹配的blastn的结果

将筛选后的blastn结果与MAGs的分类数据合并，==去除gammaproteobacteria的结果==

注意：使用tRNAscan-SE时要分别使用-A和-B两种模式分别预测、对比

9.3 全基因序列比对 - BLASTn #

使用BLASTn将viral contig与MAGs进行对比

bitscore ≥ 50, e-value ≥ 1e-3, identity ≥ 70%, and matching length ≥ 2500 bp

cat viral_MAGs_blastn.out | awk '$12 >= 50' | awk '$3 >= 70' | awk '$4 >= 2500' > viral_MAGs_blastn_filter.out

9.4 寡核苷酸比对(ONF) #

使用VirHostMatcher计算距离d2*

首先拆分viral contigs文件，每一个序列成为一个文件，并以 > 后的名称命名文件

perl -ne 'if (/^>(\S+)/) { close OUT; open OUT, ">$1.fasta" } print OUT' <viral_contigs.fasta>

运行VHM

python ~/software/VirHostMatcher/vhm.py -v viral_contigs/ -b MAGs_bin/ -o output -t host_MAGs.txt -d 1

host_MAGs.txt 为宿主MAGs分类信息，表头如下：

hostNCBIName–hostName–hostSuperkingdom–hostPhylum–hostClass–hostOrder–hostFamily–hostGenus–hostSpecies

-d 1 只输出d2*矩阵，不输出每一个的预测结果

筛选出d2*< 0.25的部分，得到viral contig与MAG的对应关系

9.5 VirHostMatcher-Net #

VirHostMatcher-Net与9.4的软件无关，其原理结合了CRISPR与ONF两种原理，使用了CRT与WIsH两个工具，自带数据库

输入文件夹，文件夹内每一个fasta文件为一个viral contig，同上，首先拆分文件，每一个序列成为一个文件，并以 > 后的名称命名文件

运行（文件夹内文件不要超过1500个，太多程序会崩）

nohup /home/zhanyuanchao/miniconda3/bin/python ~/software/VirHostMatcher-Net/VirHostMatcher-Net.py -q ~/software/VirHostMatcher-Net/test/mVCs --short-contig -o output -i tmp -n 1 -t 8 -d ~/software/VirHostMatcher-Net/data &

-q QUERY_VIRUS_DIR 包含viral contigs的文件夹，文件以.fasta、.fa名

-t NUM_THREADS 线程. Default = 1

–short-contig 对短viral contig预测宿主，WIsH需要此参数

-o OUTPUT_DIR

-n topN Number of top predictions written to the output files. All predictions will be output if there is a tie in score. Default = 1

-i INTERMEDIATE_DIR 中间过程文件夹. Default =./intermediate_res

-l GENOME_LIST Location of the file containing host NCBI genome names of interest

-d 数据库位置

9.6 VPF-Class #

参考之前的部分[[07-分类鉴定#6 3 VPF-Class]]

阈值参考membership ratio ≥ 0.3 and confidence score ≥ 0.5.

cat host_family.tsv | awk '$3 >= 0.3' | awk '$5 > 0.5' > host_family_filter.tsv

9.7 iPHoP #

iPHoP (Integrated Phage HOst Prediction) is a bioinformatic tool for computational prediction of host taxonomy from phage genomes.

iPHoP使用多种方法预测宿主，带有数据库，并可以将自己的MAGs加入到数据库中（实验性功能）进行宿主预测。

  • RaFAH：产生带有相关分数的宿主属预测

  • blastn对比MAGs基因组

  • blastn对比CRISPR spacer数据库

  • WIsH：基于病毒和宿主基因组之间k-mer组成相似性的宿主关联

  • VHM - s2* 相似性：k-mer 组成相似性的宿主关联

  • PHP：基于病毒和宿主基因组之间k-mer组成相似性的宿主关联

  1. 安装
$ conda create -n iphop python=3.8
$ conda activate iphop
$ conda install -c conda-forge -c bioconda iphop

下载数据库，gz压缩文件约109GB，解压后199G

tar -zxvf iPHoP.latest.tar.gz
# 验证数据库完整性（可选）
iphop download --db_dir path_to_iPHoP_db --full_verify
  1. 使用
# 运行时间较长，约28小时
iphop predict --fa_file my_input_phages.fasta --db_dir path/to/iphop_db/Sept_2021_pub/ --out_dir iphop_output/ --num_threads 20

实验性功能：将自己的MAGs加入到数据库中，组成新数据库进行宿主预测
