宏基因测序结果为两个read，为fastq格式文件，首先进行质检（FastQC等），查看read质量，如果是raw data，按质检结果使用质控软件质控read（Fastp等）；公司一般会直接给clean data，也要质检看下质量，按需求决定要不要质控。

1. 质控—Fastp(v0.23.0) #

$ fastp -i FFR-6_read_1.fastq -o FFR-6_clean_1.fastq -I FFR-6_read_2.fastq -O FFR-6_clean_2.fastq -e 20 -l 30 -n 0

去除质量分小于20、长度小于30、任何含有N的read

2. 质检—FastQC #

$ fastqc -t 12 FFR_6.clean.1.fastq FFR_6.clean.2.fastq

质控好的高质量reads就可以进行组装了

3. 组装—MetaSpades(进化所服务器v3.13.0-dev) #

$ spades.py --meta -1 FFR-6_clean_1.fastq -2 FFR-6_clean_2.fastq -t 16 -m 6400 -o metaspades

或者

$ metaspades.py -1 FFR-6_clean_1.fastq -2 FFR-6_clean_2.fastq -t 16 -m 6400 -o metaspades

使用-m 内存数×8。20G数据，16核，800G内存，运行约30个小时

组装好的contigs（如果用metaSPAds组装，结果最好用scaffolds文件）进行下一步病毒序列识别与筛选

4. 识别病毒序列 #

4.1.1 Virsorter(v1.0.6, 2019年10月) #

$ conda activate virsorter
$~/software/virsorter/VirSorter/wrapper_phage_contigs_sorter_iPlant.pl -f ../scaffolds.fasta --db 1 --wdir output_all --ncpu 8 --data-dir ~/software/virsorter/virsorter-data

分类为1-6的序列名提取出来 > 1_6.txt 1，2，4，5的序列名提取出来 > 1245.txt

4.2.2 DeepVirFinder(v1.0) #

$ conda activate deepvirfinder
$ nohup python ~/software/DeepVirFinder/dvf.py -i ../viromics/scaffolds.fasta -l 1500 > deepvirfinder.log 2>&1 &

-l 1500 ，不预测1500bp以下的序列

-m MODDIR, –mod=MODDIR, model directory (default ./models) -l CUTOFF_LEN, –len=CUTOFF_LEN, predict only for sequence >= L bp (default 1) -c CORE_NUM, –core=CORE_NUM, number of parallel cores (default 1)

提取score≥0.9, p value<0.05的序列名 > 09.txt

cat 1500_dvf.txt | awk '$3 >= 0.9' | awk '$4 < 0.05' > 09.txt

提取0.9<score≥0.7, p value<0.05的序列名 > 0709.txt

cat 1500_dvf.txt | awk '$3 < 0.9' | awk '$3 >= 0.7' | awk '$4 < 0.05' > 0709.txt

4.3 JGI Pipeline #

1、提取出≥5kb的序列 #

使用seqkit提取序列：

$ seqkit seq -m 5000 scaffolds.fasta > scaffolds_5000.fasta

-m 按照序列长度过滤，表示保留的最小长度，-M 此为保留的最大长度

2、使用hmmsearch对比病毒蛋白家族 #

注释前，使用prodigal(-p meta)预测ORF，以下都需要faa文件进行注释对比

$ prodigal -p meta -a protein_seq.faa -m -d nucleotide_seq.fna -i scaffolds_5000.fasta
$ hmmsearch --cpu 0 -E 1.0e-05 --tblout <OUTPUT_File> <VIR_HMM_File> <YOUR_Input_Protein_FASTA_File>

<VIR_HMM_File>为构建的病毒蛋白家族，下载链接 https://img.jgi.doe.gov//docs/final_list.hmms.gz

3、使用Pfam、KEGG注释病毒ORF #

3.1 Pfam的本地注释，使用hmmscan注释 #
$ hmmscan -o out.txt --tblout out.tbl --noali -E 1e-5 /opt/database/pfam_data/Pfam-A.hmm file.fasta

out.tbl 文件则是表格形式的结果，是一般需要的结果。

3.2 KEGG注释 #

可以使用GhostKOALA进行在线注释（文件<300Mb） 也可以使用KofamKOALA，用hmmscan本地注释，参考1 ,参考2

4、合并以上三个hit的结果，统计成主表（重要与关键!） #

统计出每个scaffold的基因数，hit到VPFs、pfam、ko的基因数（注意！对于一个基因注释结果可能有多个，注意统计结果） 表入下：

统计序列文件中每个scaffold所含有的基因数量 get_nodeNumList.py 使用R整理成上述表格

5、使用三种方法过滤筛选 #

$ cat Master_Table.txt | awk '$2 >= 5' | awk '$8 <= 20' | awk '$6 <= 40' | awk '$4 >= 10' | cut -f 1 > Filter1.out
$ cat Master_Table.txt | awk '$2 >= 5' | awk '$2 >= $5' | cut -f  1 > Filter2.out
$ cat Master_Table.txt | awk '$2 >= 5' | awk '$4 >= 60' | cut -f  1 > Filter3.out

合并三种过滤文件

$ cat Filter1.out Filter2.out Filter3.out | sort | uniq > JGI.txt

4.4 合并三种方法识别到的病毒序列（使用uniq时要注意文件编码格式需统一） #

VirSorter中1，2，4，5类与DeepVirFinder中score≥0.9, p value<0.05的取并集合在一起

cat 1245.txt 09.txt | sort | uniq > 1.txt

VirSorter中1-6类与DeepVirFinder中score在0.7~0.9, p value<0.05的取交集合在一起

cat 1_6.txt 0709.txt | sort | uniq -d > 2.txt

并集合并VirSorter与DeepVirFinder的结果

cat 1.txt 2.txt | sort | uniq > vir.txt

最后，并集合并入JGI的结果

cat vir.txt JGI.txt | sort | uniq > final_viromes_list.txt

利用以上的序列名文件，用get_seq_bylist.pl脚本从组装后的scaffolds文件中抽提出病毒基因序列

5. 病毒聚簇（BLASTn聚类） #

分别对每个样品进行BLASTn聚类，方便进行相对丰度分析，然后合并所有数据，在聚类

本地建库

makeblastdb -in <db.fasta> -dbtype nucl -out <dbname>

-in 需要制作数据库的序列文件(输入fna文件) -dbtype 输入序列的类型，核酸为nucl，氨基酸为prot -out 输出得到的数据库的名称，每个序列文件都会输出一个数据库，产生的几个文件，一般可以创建一个文件夹，将这几个文件都保存在里面

blastn比对

nohup blastn -query <viral.fasta> -db <$../vir_db/> -outfmt '6 std qlen slen' -out <viral_blastn.blout> -evalue 1.0e-50 -perc_identity 80 -num_threads 16 > allvall_blast.nohup.out 2>&1 &

去除自身对比的行

cat <viral_blastn.blout> | awk '$1 != $2' > <viral_blastn_noselfhits.blout>

筛选出identity>=90%，coverage length>=75%，covered length requires at least one contig of >500bp length

java Parse_BLAST <viral_blastn_noselfhits.blout>  ><viral_blastn_noselfhits_parsed.blout>

整理信息，聚簇

perl SLC.pl <viral_blastn_noselfhits_parsed.blout> <viral_all_groups.slc>

使用R统计出每簇最长的序列作为代表序列，将冗余序列的node号整理成一个list，方便后续去掉这些序列 R脚本 cluster_w_rep.R

6.1 vcontact2 #

1. 使用prodigal预测ORF #

prodigal -p meta -a <virome.faa> -m -d <virome.fna> -i <viral_blastn.fasta>

2. 准备gene2genome文件（protein_id, contig_id, keywords信息的csv文件） #

激活环境

conda activate vcontact2
vcontact2_gene2genome --proteins <virome.faa> --output <out_map.csv> --source-type Prodigal-FAA

–source-type {VIRSorter,Prodigal-coords,Prodigal-FAA, MetaGeneMark,NCBICodingSequence,NCBIFasta}

3. 运行程序 #

nohup vcontact2 --rel-mode 'Diamond' --pcs-mode MCL --vcs-mode ClusterONE --c1-bin ~/software/cluster_one-1.0.jar --db 'ProkaryoticViralRefSeq201-Merged' --verbose --threads 10 --raw-proteins <virome.faa> --proteins-fp <out_map.csv> --output-dir vcontact_out > out.log 2>&1 &

参数 –raw-proteins FASTA-formatted proteins file –proteins-fp A file linking the protein name and the genome names (csv or tsv) –rel-mode {BLASTP,Diamond,MMSeqs2} 蛋白比对方法，计算蛋白序列相似性 –pcs-mode {ClusterONE,MCL} 蛋白聚类方法 –vcs-mode {ClusterONE,MCL} 病毒聚类方法 –c1-bin “cluster_one-1.0.jar"的路径 –db 参考库

结果统计： 统计 genome_by_genome_overview.csv 的VC簇及VC亚簇

6.2 blastp #

1. Diamond blastp对比 #

vConTACT2分类病毒的数目有限，使用blast对比病毒数据库进行进一步分类 NCBI Viral RefSeq数据库

diamond makedb --in viral_refseq_211.faa -d viral_refseq_211 --taxonmap ~/software/ncbi_database/taxonomy/prot.accession2taxid.gz --taxonnodes ~/software/ncbi_database/taxonomy/taxdmp/nodes.dmp --taxonnames ~/software/ncbi_database/taxonomy/taxdmp/names.dmp

nohup diamond blastp -d diamond_db_5000/viral_refseq_211 -q virome_5000.faa -o viral_5000_diamond.out -f 6 qseqid sseqid evalue bitscore staxids sscinames -p 10 --subject-cover 50 --query-cover 50 --more-sensitive -e 1e-5 -k 1 &

2. 使用Taxonkit获取taxID分类信息 #

用于查找taxID对应所在的分类水平（科属种） 官网; 参考

安装

conda install taxonkit -c bioconda -y

表格数据处理，推荐使用 csvtk 更高效

conda install csvtk -c bioconda -y

下载taxonomy数据（注意taxonomy数据更新频繁，最好与上述对比时使用相同的文件）

wget -c https://ftp.ncbi.nih.gov/pub/taxonomy/taxdump.tar.gz 
tar -zxvf taxdump.tar.gz

解压文件存于家目录中.taxonkit/，程序默认数据库默认目录

mkdir -p $HOME/.taxonkit
cp names.dmp nodes.dmp delnodes.dmp merged.dmp $HOME/.taxonkit

使用 提取diamond结果的staxids号

cat taxid_list.txt | taxonkit lineage | taxonkit reformat -F -f "{o}\t{f}\t{g}\t{s}" > taxid_out.txt

3. R处理统计 #

准备文件：1）diamond结果；2）taxid分类结果；3）prodigal预测每个node的基因数量文件 按taxid号，合并diamond结果与taxid分类结果 统计node的基因数量，可注释的基因数量，分类为同一科的数量 belong to a viral family if ≥50% of the proteins were assigned to that family with a bitscore ≥50
